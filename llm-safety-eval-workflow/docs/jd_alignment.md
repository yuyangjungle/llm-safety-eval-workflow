# JD 对齐说明

岗位核心不是普通 AI 应用产品，而是大模型底层数据基建和安全评测相关的数据产品能力。本项目按 JD 四类职责拆解。

## 1. 数据策略制定

项目通过 `data/risk_taxonomy.json` 与 `docs/data_taxonomy.md` 将抽象安全议题拆成 8 类可生产数据任务，并定义每类的预期行为、风险信号和评估关注点。

## 2. 数据产线搭建

`scripts/generate_samples.py` 从人工种子样本出发，按风险类型生成模板化合成样本，并输出统一数据集。这个流程模拟真实团队中从模型训练/评测需求到数据生产链路的拆解过程。

## 3. 质量与效果评估

`scripts/evaluate_quality.py` 对样本进行 schema 完整率、风险覆盖率、去重率、rubric 完整度、难度分布等指标评估，并输出 `docs/eval_report.md`。

## 4. 模式创新

`demo/` 用静态 dashboard 展示 schema-first workflow、质量门禁和数据飞轮。后续可以扩展为接入真实 LLM 输出、LLM-as-judge、人工抽检和自动补样。
